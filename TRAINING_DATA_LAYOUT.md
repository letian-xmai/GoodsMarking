# 模型训练数据目录说明

## 业务图片目录

### 商品数据
路径：`模型训练数据/商品数据/`

含义：人工查看入口，先按商品编码分组，再放入该商品的原始照片、人工标注结果、每次模型运行结果。

结构：

```text
模型训练数据/商品数据/{outward_code}/
  商品原始照片/
  人工标注结果/
  模型运行结果/{run_id}/
    模型选中/
    模型排除/
    不匹配/
```

- `商品原始照片/`：测试集去重后的该商品全部候选图片，包含 `备选` 和 `选中`。
- `人工标注结果/`：该商品人工标注为 `选中` 的正样本图片。
- `模型运行结果/{run_id}/`：该商品某次模型评估的输出；`run_id` 默认使用运行时间，格式为 `YYYYMMDD_HHMMSS`。
- `模型选中/`：模型预测为选中的图片。
- `模型排除/`：模型预测为排除的图片。
- `不匹配/`：模型预测和人工标签不一致的图片，供人工复核。

## 图片命名规范

- `images/{train|val|test}/`、`商品数据/{outward_code}/商品原始照片/`、`人工标注结果/`、`模型运行结果/{run_id}/...` 中的图片统一使用 URL path 的原始文件名。
- URL 转义路径先解码，例如 `tagmark%2Fframe_188_1782217054_0478b5.jpg` 保存为 `frame_188_1782217054_0478b5.jpg`。
- 同一目录内出现同名 URL 文件时，后续文件追加 `_毫秒时间戳`，禁止覆盖。
- `sample_id`、`outward_code`、标签、split、人工/模型结果只保存在 JSONL 字段和目录层级中，不写进图片文件名。

## Manifest 字段

`模型训练数据/manifest_all.jsonl` 中和目录对应的字段：
- `dataset_roles`：该图片属于哪些数据角色。
- `original_image_path`：对应 `商品数据/{outward_code}/商品原始照片` 路径。
- `annotation_result_path`：正样本对应 `商品数据/{outward_code}/人工标注结果` 路径；负样本为 `null`。
- `image_path`：兼容模型训练代码的原始路径，文件名仍按 URL 原名保存。

## 固定命令

重建训练数据：

```bash
python3 -m image_workflow.cli build-training-set --label-workbook 测试集.xlsx --output-dir 模型训练数据 --download-workers 4
```

生成一次模型运行结果：

```bash
python3 -m image_workflow.cli evaluate-full-testset --dataset-dir 模型训练数据 --model-dir 模型训练数据/model --no-preview
```

## 注意事项
- `商品数据` 是人工查看入口。
- `商品原始照片` 是单个商品的全集。
- `人工标注结果` 是单个商品人工选中的正样本。
- `模型运行结果` 是单个商品每次跑出来的预测结果。
- `images/` 目录仅为兼容已有训练代码保留，不作为人工查看入口；这里可以继续保留 `train/val/test`。
- 旧顶层目录 `模型训练数据/商品原始照片/`、`模型训练数据/人工标注结果/`、`模型训练数据/模型运行结果/`、`模型训练数据/商品标注结果/` 会在重建或评估时归档到 `模型训练数据/_legacy/目录名_时间/`，当前流程不再写入这些顶层业务目录。
